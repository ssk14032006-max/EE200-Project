# Sonic Signatures — Q3B App

Matches the structure required: Library / Identify / Batch
tabs, dark+teal theme, pipeline timing breakdown, cluster score, candidate
ranking, and the three explanatory plots (constellation, full-song
fingerprint with query window highlighted, offset-histogram spike).

## Folder structure
```
q3b_app/
├── .streamlit/config.toml   # dark/teal theme
├── audio_io.py               # loads wav/mp3/flac/ogg/m4a via pydub+ffmpeg
├── fingerprint.py             # peak-picking, hashing, matching, timing
├── build_database.py          # one-time indexing script
├── app.py                      # the Streamlit app (3 tabs)
├── requirements.txt
├── packages.txt                # ffmpeg, for Streamlit Community Cloud
├── database.pkl                # generated by build_database.py -- ships with the app
├── song_library/                 # your song files (NOT shipped with the deployed app)
└── samples/                      # OPTIONAL: a few short demo clips for the "Try a sample" buttons
```

## 1. Index your songs (run once, locally)

Put your provided songs (any of .wav/.mp3/.flac/.ogg/.m4a) into a
`song_library/` folder next to these scripts, then:

```bash
pip install -r requirements.txt
# also need ffmpeg on your system: e.g. `sudo apt install ffmpeg` (Linux),
# `brew install ffmpeg` (Mac), or download a build for Windows
python build_database.py
```

This creates `database.pkl`. Redo this only if you add/remove songs or
change the hashing parameters in `fingerprint.py`.

## 2. (Optional) Add sample clips

Drop a handful of short (~10-30s) clips into a `samples/` folder to power
the "OR TRY A SAMPLE" buttons in the Identify tab. Not required -- the
upload button works fine on its own -- but it makes the demo nicer to
click through without needing your own files on hand.

## 3. Run locally to test

```bash
streamlit run app.py
```

Check all three tabs:
- **Library** — thumbnails + hash counts render for every indexed song.
- **Identify** — upload (or pick a sample), hit Identify, confirm the
  pipeline timing, match banner, candidate bars, and all three step plots
  render correctly.
- **Batch** — upload several clips, run, confirm `results.csv` downloads
  with exactly `filename, prediction` columns and the right summary line.

## 4. Deploy on Streamlit Community Cloud

1. Push this folder to a GitHub repo, **including `database.pkl`** (and
   `samples/` if you're using it). Do NOT push `song_library/` -- the
   raw audio isn't needed by the deployed app, only the index is.
2. Go to share.streamlit.io → New app → select repo/branch → main file
   `app.py` → Deploy. `requirements.txt` and `packages.txt` are picked
   up automatically (the latter installs ffmpeg, needed for mp3/m4a).
3. Copy the live app URL + your GitHub repo link for the report.

## Notes
- `MATCH_THRESHOLD` in `fingerprint.py` should be set to whatever value
  you determined empirically in your notebook's robustness testing (the
  score that separates real matches from noise).
- `database.pkl` stores per-song peak arrays for the Library thumbnails
  and the Step 2 "full fingerprint" plot, so it's larger than a bare
  hash table -- if it gets too big for a normal GitHub push, use Git LFS.
- If `database.pkl` was built before this update (the old format was a
  flat hash dict, not `{'hashes':..., 'songs':...}`), rebuild it with the
  current `build_database.py` -- the app expects the new structure.
